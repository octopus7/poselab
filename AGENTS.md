# AGENTS.md

이 저장소는 CodexPose 정적 웹 뷰어와 모션 의도 문서를 함께 관리한다.

## 작업 원칙

- 사용자가 만든 변경을 되돌리지 않는다. 기존 변경이 작업과 겹치면 먼저 읽고 그 상태에 맞춰 수정한다.
- 정적 페이지 코드는 `web/`에 둔다.
- 모션의 의도, 키 타이밍, 접지 기준, 후속 수정 주의점은 `docs/`의 별도 Markdown에 기록한다.
- 모션 생성 요청은 항상 keypose 기반으로 구현한다. 모션별 원본 의도와 주요 자세는 `sourceKeys`/`keyposes` 데이터로 남기고, 뷰어는 그 데이터를 읽어 재생해야 한다.
- UI를 바꿀 때는 데스크톱과 모바일에서 텍스트가 컨테이너 밖으로 넘치지 않는지 확인한다.

## 병렬 작업 지침

조사, 구현, 검증이 서로 독립적이고 병렬 수행이 가능한 경우에는 `spawn_agent`로 에이전트를 스폰해서 나누어 수행한다. 이때 각 에이전트에는 담당 파일과 책임 범위를 명확히 지정하고, 서로의 변경을 되돌리지 않도록 안내한다. 결과를 합치기 전에는 변경 파일 목록과 동작 영향을 확인한다.
