# Walk / Run Motion Intent

이 문서는 `web/index.html`의 걷기/뛰기 모션 의도를 나중에 다시 파악하기 위한 기록이다. 참고한 정적 페이지는 `D:\github\ue5_codex\CodexPose\web`의 Three.js 기반 Manny 뷰어이며, 현재 구현은 같은 성격의 정적 웹 미리보기로 축소했다.

## 공통 기준

- 기준 캐릭터: UE5 Manny 비율을 의식한 단순 3D 스켈레톤
- 좌표계: X는 좌우, Y는 위, Z는 진행 방향
- 표현 방식: 루트는 중앙에 두고 양발을 러닝머신 방식으로 앞뒤 이동시켜 루프를 읽기 쉽게 표시
- 색상: 왼쪽은 파랑, 오른쪽은 주황, 중앙 체인은 어두운 회색, 접지 발은 초록
- 조작: 모션 선택, 재생/일시정지, 정지, 프레임 타임라인, 속도 조절, 전면/측면/상단 시점

## 걷기 모션

- ID: `walk`
- 길이: 48프레임
- 재생 속도: 24fps
- 의도: 안정적인 보행 루프. 달리기처럼 보이지 않도록 양발 접지 구간을 길게 두고 체공을 만들지 않는다.
- 발 타이밍: 한 발의 stance 비율을 약 62%로 설정해 뒤꿈치 닿기, 체중 받기, toe-off, 스윙 복귀가 겹쳐 보이게 한다.
- 루트/골반: 상하 바운스는 작게, 좌우 무게 이동은 분명하게 둔다. 골반은 지지발 쪽으로 살짝 이동한다.
- 상체: 앞으로 살짝 기울이되 과장하지 않는다. 가슴은 골반과 반대 방향으로 약하게 카운터 회전한다.
- 팔: 다리와 반대 위상으로 낮고 자연스럽게 흔든다. 손은 어깨 아래에 유지한다.
- 다리: 스윙 발은 낮게 들어 올리고, 무릎은 과하게 접지 않는다.

## 뛰기 모션

- ID: `run`
- 길이: 32프레임
- 재생 속도: 32fps
- 의도: 빠른 전방 진행감을 가진 자연스러운 뛰기 루프. 걷기와 구분되도록 체공, 강한 압축, 팔 펌프를 명확히 한다.
- 발 타이밍: 한 발의 stance 비율을 약 38%로 줄여 양발이 모두 공중에 있는 구간을 만든다.
- 루트/골반: 착지 때 루트를 낮추고, 밀어내기 이후 체공 구간에서 높인다.
- 상체: 걷기보다 전경사를 크게 둔다. 가슴과 어깨는 골반 반대 방향으로 더 강하게 틀어 추진감을 만든다.
- 팔: 팔꿈치를 굽힌 상태에서 강하게 앞뒤로 펌프한다. 손은 걷기보다 몸 가까이, 더 높은 위치로 지나가게 한다.
- 다리: 스윙 발은 높게 접고, 앞발 리치는 길게 둔다. 착지 발은 초록 접지 링으로 표시한다.

## 후속 수정 시 주의점

- 모션 수치를 바꿀 때는 `web/motion_viewer.js`의 `MOTIONS.walk`, `MOTIONS.run` 값을 먼저 조정한다.
- 새 모션을 추가하면 이 문서에 ID, 프레임 수, fps, 발 접지 의도, 상체/팔/다리 의도를 함께 기록한다.
- 실제 애니메이션 에셋으로 옮길 때는 발 접지 프레임과 루트 높이 변화부터 고정하고 팔과 상체 보정은 그 다음에 맞춘다.
